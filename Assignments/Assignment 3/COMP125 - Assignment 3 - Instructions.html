What is Mad Libs?

Mad Libs is a word game where one player prompts another for a list of words to substitute for blanks in a story. The game is typically played by asking for specific types of words (nouns, adjectives, verbs, etc.) without revealing the context. Once all words are collected, they’re inserted into the story, often creating humorous or nonsensical results. For this assignment, you’ll be creating a digital version where users input their words through a form, validate those inputs, and then display the completed story.

Objective

Your task is to complete a Mad Libs game using JavaScript. You will:

  • Implement form validation using regular expressions (regex).
  • Dynamically update the embedded story with user input.
  • Ensure proper error handling and user experience improvements.

You are only allowed to edit the script.js file.
The index.html and styles.css files are provided to understand the structure and layout.

Requirements

  1. Form Validation (15%)

    • Each input field must be validated using JavaScript and regex.
    • If invalid, an error message appears below the field. (Error messages can be seen on the example link above).
    Field Validation Rule Example
    Student Name Must start with a capital letter, only letters "Sam"
    Professor Name Must start with a capital letter, only letters "Dr. Johnson"
    Adjective At least 5 characters long "exciting"
    Verb (-ing form) Ends in -ing "jumping"
    Number Between 1-100 "25"
    Phone Number Format: 123-456-7890 "647-555-1234"
    Cartoon Character Any text (required) "Bugs Bunny"
    Song Title Any text (required) "Bohemian Rhapsody"
    Colour Selected from dropdown "Blue"

    ⚠️ The submit button must remain disabled until all fields are valid.

  2. Dynamic Story Generation (15%)

    • Once the form is submitted:
      • The input values replace placeholders in the story.
      • The story replaces the form on the page.
      • The background colour changes based on the user’s selection.

Hints

  • Use the submit event on a form for the event listener.
  • Use e.preventDefault() to stop the form from submitting using the HTML default behavior.
  • The story is already in the HTML; you need to replace each span in the story with the form input values.
  • Use form.reset() to clear inputs when resetting.
  • Use regular expressions (regex) with the test() method to validate each input field against the specified patterns.
  • Consider using the disabled attribute on the submit button and toggle it based on form validation results.
